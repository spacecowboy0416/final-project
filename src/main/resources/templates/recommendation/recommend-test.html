    입력 제약
    
                
      
                    
    • AI 입력에는 이미지 1장, 자연어 요청, 대상 성별, 일정 시각이 포함됩니다.
      • 
                    
    • location은 서버가 날씨를 조회할 때만 쓰는 요청 필드입니다.
      • 
                    
    • 날씨 요약은 서버 내부에서 붙고, 이 테스트 페이지에서는 별도 설명 블록으로 노출하지 않습니다.
      • 
                
    
                
    
                    input json schema
                    
    {
  "naturalText": "...",
  "gender": "male",
  "scheduleTime": "...",
  "location": {
    "districtName": "중구",
    "placeName": "서울특별시청",
    "addressName": "서울특별시 중구 세종대로 110",
    "latitude": 37.5665,
    "longitude": 126.978
  },
  "imageData": {
    "imageBytes": "base64",
    "mimeType": "image/jpeg"
  }
}
    
                
    
            
    

            
    
                
    출력 제약
    
                
    
                    output json schema
                    
    {
  "ai_blueprint": {
    "tpoType": "date",
    "styleType": "minimal",
    "main_item_analysis": {
      "temp": "mild",
      "season": "spring",
      "color": "gray",
      "type": "top",
      "style": "minimal"
    },
    "coordination": {
      "tops": {
        "item_name": "...",
        "category": "top",
        "attributes": {
          "color": "gray",
          "material": "cotton",
          "fit": "oversized",
          "style": "minimal"
        },
        "temp_range": [10, 18],
        "reasoning": "...",
        "priority": "essential"
      }
    },
    "styling_rule_applied": "..."
  }
}
    
                
    
            
    

            
    
                
    의미 규칙
    
                
      
                    
    • tpoType은 상황 코드이고 스타일 코드가 아닙니다.
      • 
                    
    • styleType과 attributes.style은 스타일 코드입니다.
      • 
                    
    • item_name은 사용자 노출용 이름입니다.
      • 
                    
    • 쇼핑용 search_query는 AI 출력이 아니라 서버 비즈니스 로직에서 속성값으로 조합합니다.
      • 
                    
    • reasoning은 각 슬롯이 전체 코디에 왜 맞는지 짧게 설명해야 합니다.
